# 更新记录

[返回中文首页](../README.md) · [完整使用指南](使用指南.md) · [English](../README.en.md)

这里按功能阶段记录项目的重要变化。它不是逐条 Git commit 清单，而是帮助使用者理解“当前版本比早期版本多了什么”。

## 2026-07-17：扩展 YouTube 视频地址兼容性

- YouTube Shorts、Live、Embed、旧式 `/v/` 和 `/e/` 地址可直接提取视频 ID 并进入下载流程。
- 增加 Music YouTube、Gaming YouTube 和 `youtube-nocookie.com` 嵌入地址识别。
- 保留普通 `watch?v=` 与 `youtu.be` 地址兼容性；视频删除、私有、会员、地区或登录限制仍由 YouTube 和 `yt-dlp` 判断。

## 2026-07-17：可替换画面字幕 OCR

- 新增平台无关的画面 OCR 引擎协议和注册表，主流程不再绑定某一个 OCR 实现。
- 首个引擎使用 macOS 系统 Vision；不需要下载额外 OCR 模型，编译后的辅助程序会在本地缓存复用。
- 自动字幕来源在 Whisper 结果为空或高度重复时切换画面 OCR，避免无对白视频生成整段重复假字幕。
- CLI 新增 `--source screen-ocr`，Web 字幕来源新增“画面字幕 OCR”，环境区显示引擎可用状态。
- OCR 结果按位置、置信度和文字特征过滤，并对连续帧的相似文字做时间轴合并。
- Linux 和 Windows 不会因为 Apple Vision 缺失而无法启动；后续可按统一接口接入其它本地或云端 OCR 引擎。

## 2026-07-16：低风险翻译提速

- z.ai 在一个目标语言最终失败后，会在当前任务内熔断，后续语言直接进入回退链路，避免每种语言重复等待超时。
- NLLB 和 z.ai 每次保存翻译批次时同步更新任务总百分比，并显示当前语言的已完成字幕数量。
- 完全相同的源字幕在单次任务中合并翻译，完成后按原索引展开，不改变原时间轴。
- 翻译断点仍保存完整时间轴索引，继续任务和现有翻译缓存保持兼容。

## 2026-07-16：阶段二配置精简与阶段三 NLLB 稳定性

- 翻译引擎标识、兼容别名、显示名称和缓存身份统一由一个配置模块维护。
- Web 环境检查从服务主模块拆出，降低页面服务与模型检测之间的耦合。
- 目标语言目录改为独立前端资源，避免主交互脚本长期堆积语言配置。
- Web 常用设置保持直接可见，模型路径、字幕视频、编码、位置和性能开关集中到可折叠的“高级设置”。
- NLLB 1.3B 根据设备与系统内存自动选择批量，内存不足时自动减半批量并重试。
- NLLB 日志显示批次完成数、已用时间和预计剩余时间，减少长任务“像卡住”的不确定感。
- NLLB 每批保存部分结果，失败、停止或重启服务后继续任务时可跳过已经完成的字幕。
- 旧参数 `local-nllb` 与当前 `local-nllb-quality` 使用相同的 1.3B 缓存身份，升级后不重复翻译。

## 2026-07-16：本地多语言模型精简

- Web 页面、本地环境检查和当前帮助文档不再提供 NLLB 600M入口。
- 本地多语言翻译统一使用 NLLB 1.3B，减少模型选择并提高默认翻译质量。
- z.ai 失败后的本地多语言回退改为 NLLB 1.3B。
- 旧 CLI 参数 `local-nllb` 和历史任务继续兼容，但实际也使用1.3B。
- 已经下载的600M缓存不会自动删除，避免未经用户确认清理本地文件。

## 2026-07-15：阶段一可靠性优化

- 本地 Whisper 改为按实际字幕结果判断成功，命令退出正常但输出为空时也会自动降级。
- 转写降级链路统一为 `Metal + VAD -> CPU + VAD -> CPU 关闭 VAD`。
- VAD 只识别出（笑）、环境音等声音标记时，不再误判为有效对白，会关闭 VAD重新转写。
- 视频网址解析、下载、本地转写和硬字幕烧录增加总超时与心跳日志。
- 硬字幕烧录增加无进度超时，默认连续 5 分钟没有 FFmpeg 输出便停止异常进程。
- 常见下载、限流、模型缺失、空字幕和超时错误增加中文处理建议，同时保留技术信息便于排查。

## 2026-07-14：文档整理与本地数据管理

- 中文 README 改为项目主入口，英文简介独立放在 `README.en.md`，完整操作和阶段更新分别归入中文使用指南与更新记录。
- 公共界面和文档统一使用“公开视频网址”描述，内部专用站点适配能力保持不变。
- 任务历史增加“清空历史”，只删除已完成、失败、停止和中断的记录，排队和运行中的任务会保留。
- 缓存管理增加“清空全部”，同时保留按视频、音频、字幕、转写、翻译和画面分析分类清理。
- 所有清理操作执行前都会显示确认框，说明删除范围和保留内容。
- 清理任务历史或缓存均不会删除 `output/` 中已经生成的视频和字幕。

## 2026-07-13：性能与任务安全

- 本地 Whisper 默认尝试 Metal/GPU 加速，不兼容时自动切换 CPU。
- 加入 Silero VAD，可跳过静音片段；模型缺失或失败时自动使用标准转写。
- 硬字幕支持自动推荐、Apple VideoToolbox、快速 CPU 和高画质 CPU 四种编码策略。
- 硬字幕日志开始显示真实百分比、已处理时长、编码速度和预计剩余时间。
- 停止任务时可以终止当前 `ffmpeg`、`yt-dlp` 和 `whisper-cli` 子进程。
- Web 前端在提交后立即锁定“开始任务”，后端使用原子检查阻止重复任务。
- 页面刷新或局域网其它设备打开页面时，可以恢复当前活动任务的日志和进度。

详细说明见[性能与任务安全更新](2026-07-13-performance-and-task-safety.md)。

## 2026-07-12：多语言质量与字幕校对

- 新增 NLLB 1.3B 质量模式，与 NLLB 600M 快速模式并存。
- NLLB 支持按批次翻译、进度日志、异常句检测和单句重试。
- 本地翻译结果会检查空白、异常长度、重复短语和目标语言特征。
- z.ai 连续限流或失败后，可自动尝试本地模型；本地质量不合格时继续尝试 OpenAI。
- 翻译结果和分批结果加入缓存，中断后继续任务时可复用已经完成的部分。
- 新增浏览器字幕预览与校对：视频播放同步高亮字幕，可修改文字和开始/结束时间。
- 保存字幕时自动创建备份，可直接提交重新生成视频任务。
- 视频预览使用 HTTP Range 分段读取，避免一次加载完整视频。

## 2026-07-11：字幕位置、任务历史与缓存

- 新增多帧画面文字区域检测，用于判断原字幕更可能位于顶部还是底部。
- 新增稳定硬字幕、自动避让、底部上方、底部和顶部等位置选择。
- 字幕写出前自动换行、拆分长句并修复相邻时间重叠。
- 新增 SQLite 任务历史，服务重启后仍能查看任务。
- 失败、取消或中断的任务可以继续执行，并关联原任务记录。
- 视频、音频、内置字幕、转写、翻译和位置分析加入分类缓存。
- Web 页面可以查看缓存占用并按类别清理。
- 每次任务使用独立时间戳目录，避免多次生成的文件混在一起。

## 2026-07-10：本地多语言翻译与目标语言选择

- 增加本地中、日、英快速翻译模型。
- 增加 NLLB 600M 多语言本地翻译。
- Web 环境区会显示本地模型是否已安装，并为缺失模型给出下载命令。
- 目标语言改为下拉复选选择，支持一次生成多种语言。
- 切换翻译引擎时，目标语言选择会自动收敛到当前引擎支持或推荐的范围。

## 2026-07-09：Web 工作流、z.ai 与视频网址输入

- 从 CLI 工具扩展为本地 Web 应用，提供任务进度、日志、环境检查和输出列表。
- 增加网页上传，本地上传文件优先于 URL 输入。
- 增加 z.ai 翻译及 `.env` 配置。
- 增加公开分享链接的专用下载适配能力。
- 增加 YouTube 和 Bilibili 专用下载逻辑。
- 增加其它公开单视频网址的 `yt-dlp` 尽力解析。
- 支持输出外挂 SRT 和带默认软字幕轨的 MP4。

## 当前版本定位

当前项目已经形成完整的本地字幕工作流：

1. 准备本地视频或尝试下载公开视频。
2. 读取内置字幕、用 Whisper 从音频生成源字幕，或从画面硬字幕执行 OCR。
3. 使用云端或本地模型翻译一种或多种目标语言。
4. 检查翻译质量并在必要时切换引擎。
5. 输出 SRT、软字幕视频或固定位置硬字幕视频。
6. 在浏览器中预览、校对字幕并重新生成视频。

后续版本应优先完善产品化安装、用户权限、任务队列和更精细的字幕校对体验，而不是继续增加彼此重复的翻译入口。

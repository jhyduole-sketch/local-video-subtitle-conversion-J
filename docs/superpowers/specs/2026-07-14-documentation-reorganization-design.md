# 项目文档整理设计

## 目标

将当前集中在单个 `README.md` 中的大量内容整理为中文优先、英文辅助的分层文档。新用户应能在几分钟内理解项目用途并启动 Web 页面，进阶用户仍能找到完整配置、CLI 参数、模型说明和故障排查内容。

本次只调整文档，不改变程序行为，不提交或推送 Git。

## 文档结构

### `README.md`

作为中文项目首页，控制信息密度，重点包含：

- 一句话项目介绍和适用场景。
- 当前能力概览。
- 处理流程图。
- 系统架构图。
- 最短安装与启动步骤。
- 常见工作流示例。
- 输出文件说明。
- 文档导航、当前限制和更新摘要。

详细参数不在首页重复展开，统一链接到中文使用指南。

### `docs/使用指南.md`

承接当前 README 的完整中文内容，包含：

- 环境和依赖安装。
- API Key 与 `.env` 配置。
- Web 界面说明。
- CLI 命令和参数。
- 转写及翻译模型选择。
- 软字幕、硬字幕与位置避让。
- 缓存、任务历史和输出目录。
- 字幕预览、编辑与重新生成。
- 常见问题、限制和验证方式。

### `README.en.md`

作为英文辅助入口，不与中文使用指南完全镜像。包含：

- Project overview.
- Main features.
- Requirements and quick start.
- Basic CLI example.
- Output overview.
- Links to the Chinese full guide and release notes.

### `docs/更新记录.md`

按时间倒序整理重要功能阶段，不复制逐个 Git commit。首版覆盖：

- 性能与任务安全。
- NLLB 1.3B、多语言质量检查和失败重试。
- 字幕预览校对、画面位置检测和硬字幕避让。
- 通用 URL 下载、任务历史和缓存。
- Web UI、z.ai 和本地翻译模型的早期能力。

现有 `docs/2026-07-13-performance-and-task-safety.md` 保留，作为某次更新的详细技术说明，并从更新记录链接过去。

## 图示

README 使用 GitHub 可直接渲染的 Mermaid，不依赖外部图片服务。

### 处理流程图

展示：本地文件或 URL输入 -> 下载/读取视频 -> 内置字幕或 Whisper -> 翻译与质量检查 -> SRT -> 软字幕或硬字幕视频 -> 浏览器预览和校对。

### 系统架构图

展示：Web/CLI 入口、任务调度、媒体处理、转写引擎、翻译引擎、质量保护、缓存与输出目录之间的关系。

## 内容原则

- 中文是默认阅读路径；英文文档只承担国际用户的快速理解和启动。
- README 先讲价值和使用，再讲实现细节。
- 所有命令以当前源码运行方式为准，不虚构安装包或尚未实现的功能。
- 清楚区分已实现能力、尽力支持的通用下载和当前限制。
- 对 z.ai、OpenAI、本地快速模型和 NLLB 的定位保持客观。
- 不暴露 `.env` 中的真实密钥和本地隐私数据。

## 验收标准

- README 首屏能说明项目是什么、输入什么、输出什么。
- 中文用户可从 README 在三步内启动 Web UI。
- 两张 Mermaid 图语义清楚并可在 GitHub 渲染。
- 原 README 中仍有效的配置和故障排查内容可在新文档中找到。
- `README.md`、`README.en.md`、中文使用指南和更新记录互相链接正确。
- 文档描述与当前 CLI 参数、Web 功能及项目目录保持一致。

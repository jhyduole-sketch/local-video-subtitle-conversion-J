# 路线二：字幕排版与稳定显示设计

## 目标

解决翻译字幕过长、相邻字幕时间重叠，以及新字幕与原画面硬字幕占用同一区域的问题。继续保留无需重新编码的软字幕输出，同时增加位置可控、播放器表现一致的硬字幕视频。

## 范围

- 所有目标字幕输出前统一整理换行和时间轴。
- 单条画面字幕最多两行；超出容量时拆成连续字幕片段，并按文本长度分配原时间段。
- 相邻字幕保持最少 40 毫秒间隔，修复源字幕本身存在的时间重叠。
- 新增字幕视频模式：`soft` 保留原视频流，`hard` 使用 ASS 样式烧录并重新编码。
- 新增字幕位置：`auto`、`bottom`、`above-bottom`、`top`。
- `auto` 在启用避免遮挡时等同于 `above-bottom`，否则等同于 `bottom`。
- 硬字幕默认使用白字、黑色描边和安全边距；`above-bottom` 为原底部硬字幕预留空间。
- Web 和 CLI 均暴露视频模式与字幕位置选项，并在软字幕模式下提示播放器可能忽略位置。

## 不在本路线处理

- 不使用 OCR 判断画面上是否真的存在硬字幕。
- 不擦除、模糊或覆盖原画面硬字幕。
- 不保证 MP4 `mov_text` 软字幕在所有播放器中的位置一致。
- 不改变 z.ai、OpenAI 或本地模型的翻译语义。

## 数据流

翻译完成后先用源时间轴替换文本，再经过字幕排版器：清理空白、按显示宽度换行、必要时拆分片段、修复时间重叠并重新编号。整理后的片段写入目标 SRT。

软字幕模式继续把整理后的 SRT 封装为 `mov_text`。硬字幕模式把同一批片段渲染为 ASS，再由 FFmpeg 烧录到视频，输出 `*.fixed-sub.mp4`。

## 错误处理

- 字幕位置或视频模式参数非法时，在处理开始前返回明确错误。
- FFmpeg 缺少 ASS 过滤器或编码失败时，只把该语言的视频结果记为失败，已经生成的 SRT 保留。
- 字幕片段持续时间不足以安全拆分时保留两行文本，不截断内容。
- 取消任务时终止正在运行的硬字幕 FFmpeg 子进程。

## 验证

- 单元测试覆盖中英文显示宽度、最多两行、长句拆分、时间重叠修复和 ASS 位置样式。
- 媒体测试验证硬字幕 FFmpeg 参数、音频复制和取消回调。
- Pipeline 测试验证整理后的 SRT、软/硬输出分流及失败隔离。
- Web/CLI 测试验证默认值和参数传递。

# 画面字幕位置检测与字幕编辑设计

## 目标

让“自动避让”根据视频抽帧分析选择新字幕位置，并允许用户在 Web 页面修改生成后的 SRT 文本与时间轴，再使用编辑结果重新生成字幕视频。

## 画面检测

- 仅在稳定硬字幕、自动位置、自动避让同时启用时运行。
- 使用 FFmpeg 在视频时间轴上均匀抽取最多 8 张缩小后的灰度边缘图。
- 分别计算顶部与底部区域的横向文字边缘密度、连续行峰值和多帧命中率。
- 输出 `top`、`bottom`、`none` 或 `unknown`，并附带置信度和采样数量。
- 检测到底部字幕时，新字幕放在原字幕上方；检测到顶部字幕时，新字幕放到底部。
- 无字幕时放底部；不确定时保守放在原底部字幕上方。
- 检测结果按视频指纹缓存，重复任务不重新抽帧。
- 日志记录检测位置、置信度和最终字幕位置。

## 字幕编辑

- 输出区的目标 SRT 提供“编辑字幕”操作。
- 编辑器显示序号、开始时间、结束时间和文本；支持保存与关闭。
- 保存时后端重新解析并验证：路径必须位于用户指定输出目录、时间格式合法、结束时间晚于开始时间。
- 保存后重新执行字幕排版和时间轴防重叠，并覆盖当前 SRT，同时保留同目录 `.backup.srt`。
- 提供“保存并重新生成视频”，创建新的后台任务，沿用原视频、字幕模式和位置，输出带 `.edited` 标记的新视频。
- 不修改已经生成的视频，也不删除旧文件。

## 安全与错误

- API 仅允许访问解析后的输出目录内部 `.srt` 文件和视频文件。
- 单次编辑最多返回 3000 条字幕，避免页面失控。
- 字幕保存或重新生成失败时保留原文件和已生成结果。
- 重新生成视频使用现有单任务队列、取消机制与日志。

## 验证

- 纯函数测试覆盖顶部、底部、无字幕和低置信度检测。
- Pipeline 测试覆盖检测缓存与自动位置映射。
- API 测试覆盖路径越界、读取、保存、备份和重新生成任务。
- 浏览器验证编辑器加载、保存按钮、时间字段与错误提示。

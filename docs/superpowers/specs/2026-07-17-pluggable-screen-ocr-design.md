# 可替换画面字幕 OCR 设计

## 目标

当视频没有内置字幕、音轨没有有效对白，但画面中存在硬字幕时，自动读取画面文字并生成源字幕；同时阻止 Whisper 重复幻觉继续进入翻译流程。

## 字幕来源流程

`auto` 模式按以下顺序处理：

1. 内置字幕轨。
2. 音频语音识别。
3. 若语音识别失败或结果高度重复，切换画面 OCR。
4. OCR 不可用或没有识别到文字时，停止任务并给出明确建议。

新增显式 `screen-ocr` 来源，允许用户直接跳过音频识别。

## 引擎边界

定义统一的 `ScreenOcrEngine` 协议：

- `engine_id`：稳定缓存标识。
- `label`：日志和界面显示名称。
- `availability()`：返回可用状态及原因。
- `recognize_video()`：输入视频、源语言、取消检查和进度回调，输出 `SubtitleSegment`。

注册表负责选择引擎。第一版实现 `MacVisionOcrEngine`；Linux、Windows 或云端 OCR 可以在以后实现同一协议，无需修改字幕主流程。

## macOS Vision 实现

- 使用 FFmpeg 按固定间隔抽取缩放后的 JPEG 帧。
- Swift 辅助程序调用 Apple Vision `VNRecognizeTextRequest`。
- Swift 程序首次使用时编译到本地工具缓存，后续任务复用。
- OCR 结果通过 JSON Lines 返回文字、置信度和位置。
- Python 侧过滤低置信度、边缘台标和过小文字，并把连续相同文字合并为 SRT 时间段。
- OCR 源字幕按视频指纹和引擎标识缓存。

## 质量保护

语音字幕满足以下任一条件时判定为可疑：

- 至少 8 段字幕中，同一文本占 60% 以上。
- 至少 8 段字幕中，规范化后的唯一文本比例不超过 25%。

`auto` 模式会转入 OCR；显式 `audio` 模式会停止并提示改用画面 OCR，不再翻译错误源字幕。

## 平台行为

- macOS 且存在 `xcrun`：使用 Apple Vision。
- 其他平台：OCR 检查显示为可选能力不可用，普通内置字幕和音频流程不受影响。
- 明确选择画面 OCR 但没有可用引擎：任务快速失败并说明当前服务器需要配置 OCR 引擎。

- Pipeline 测试自动回退、显式 OCR、缓存和不可用错误。

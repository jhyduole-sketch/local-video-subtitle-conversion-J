# 2026-07-13 性能与任务安全更新

本次更新集中解决本地语音转写、硬字幕烧录进度不透明，以及 Web 页面重复提交任务的问题。

## 本地 Whisper

- 默认尝试使用 whisper.cpp 的 Metal/GPU 加速。
- Metal 运行失败时自动切换 CPU，并在当前服务进程中记住失败状态，避免后续任务反复尝试不兼容路径。
- 支持 Silero VAD 跳过静音片段；模型缺失或运行失败时自动退回标准转写。
- Web 和 CLI 均可关闭 GPU 或 VAD，也可以指定自定义 VAD 模型路径。
- 转写缓存键包含 GPU/VAD 配置，避免错误复用不同转写策略生成的字幕。

## 硬字幕烧录

- 新增四种编码方式：自动推荐、Apple VideoToolbox、快速 CPU、高画质 CPU。
- 自动模式优先尝试 Apple VideoToolbox，失败时自动切换 `libx264 veryfast`。
- 硬字幕日志显示真实百分比、已处理时长、编码速度和预计剩余时间。
- 取消任务时可以终止正在运行的 FFmpeg 子进程。
- CLI 新增 `--hard-subtitle-encoder`，Web 页面新增对应选择项。

## Web 任务防重复

- 点击“开始任务”后立即禁用按钮，并显示“任务进行中”。
- 请求尚未返回时，连续点击不会再次发送任务。
- 后端在锁内原子检查活动任务；第二个提交返回 HTTP `409` 和当前任务信息。
- 页面刷新或局域网其它设备打开页面时，会自动恢复当前任务、日志、进度和停止按钮。
- 状态轮询暂时失败时保持任务锁定并自动重试，避免误以为任务结束后重复提交。
- 主转换、失败任务继续执行和字幕编辑后重新生成视频都遵守同一活动任务限制。

## 局域网使用

本机访问：

```bash
env PYTHONPATH=src python3 -m subtitle_tool.web --host 127.0.0.1 --port 7860
```

局域网访问：

```bash
env PYTHONPATH=src python3 -m subtitle_tool.web --host 0.0.0.0 --port 7860
```

局域网设备通过运行服务电脑的 IP 和端口访问，例如 `http://192.168.1.20:7860/`。该模式目前没有登录认证，只适合可信网络。
